## Change Summary

WP-103 adds an opt-in, diagram-only Planner metaphor supplement.

- `ExpConfig.planner_metaphor` defaults to `False`.
- `main.py` and `skill/run.py` expose `--planner-metaphor`.
- `agents/planner_agent.py` uses `build_planner_system_prompt(...)` so the
  default diagram prompt remains unchanged and plot prompts ignore the flag.
- The supplement instructs the Planner to use a compact visual metaphor only as
  a conceptual composition guide before producing the same detailed textual
  description output.
- No SVG/Cairo renderer, new dependency, or default prompt behavior is added.

## Limitations

- No live provider generation was run with metaphor mode enabled.
- No benchmark or human review has shown that metaphor mode improves output
  quality; it remains opt-in and default-off.
- This evidence validates integration on the Python branch only. Native UI
  exposure, if desired, remains a later native integration/product decision.
