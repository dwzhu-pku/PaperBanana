# WP-007 Reference Example Row AX Evidence

## Summary

This evidence records a bounded live Accessibility probe of native Prompt Studio
manual reference example rows. It verifies that the installed Release app exposes
reference rows, selection summary state, selected row state, search-field state,
and selection-cap disabled state through the macOS Accessibility tree.

The probe did not press `Generate`, did not press `Dry Run`, did not open a
preflight sheet, did not create a run folder, and did not use provider
credentials.

This advances WP-007/T-021 for reference-example row traversal, but it does not
close full manual VoiceOver speech-output traversal, running-disabled row state,
Settings Increased Text Size, broader hover/focus/adaptive visual evidence,
live provider E2E, hosted validation, quality benchmarking, or release/rollback
gates.

## Setup

The app was launched into Prompt Studio with the local PaperBananaBench-backed
repository:

```bash
defaults write local.paperbanana.gui settings.repoPath -string "/Users/jeff/Codex_projects/PaperBanana"
defaults write local.paperbanana.gui paperbanana.intent.destination -string promptStudio
defaults write local.paperbanana.gui paperbanana.intent.prompt -string "WP-007 reference row AX validation prompt."
open -n -b local.paperbanana.gui
```

Accessibility state was queried with transient Swift scripts using
`AXUIElementCreateApplication`. The probe used native keyboard input to drive the
`Search examples` field after direct AX value injection was found not to update
the SwiftUI binding.

## Initial Row State

The reference panel and visible rows exposed these initial live AX states:

| Element | Observed AX state |
|---|---|
| `reference-examples-panel` | `AXGroup`, description `Reference examples`, enabled `true` |
| `reference-examples-selection-summary` | values `0/10` and `No manual examples selected.` |
| `workspace-search-search-examples` | `AXTextField`, description `Search examples`, value `No search text`, help `Search examples` |
| `reference-example-ref_1` | `AXButton`, description `Reference example ref_1`, enabled `true`, help `Select this reference example`, value begins `Not selected. Image available.` |
| `reference-example-ref_2` through `reference-example-ref_6` | visible, enabled selectable row buttons with `Not selected. Image available.` values |

After pressing `reference-example-ref_1`, the row and summary updated:

```text
reference-examples-selection-summary value=1/10
reference-example-ref_1 enabled=true selected trait present
reference-example-ref_1 help=Remove this reference example
reference-example-ref_1 value begins Selected. Image available.
reference-example-ref_2 remains Not selected. Image available.
```

## Search And Selection Cap Results

Keyboard-driven search was validated after focusing the native search field. A
search for `ref_7` changed the search field value and showed the matching row:

```text
workspace-search-search-examples value=ref_7
reference-example-ref_7 value begins Not selected. Image available.
reference-example-ref_1 missing from the filtered visible tree
```

The probe then selected ten examples without starting a run. Selected ids were:

```text
ref_1
ref_2
ref_3
ref_4
ref_40
ref_41
ref_42
ref_43
ref_44
ref_45
```

The exact ids are not semantically important; the validation target was the live
row state transition and 10-example cap. The `ref_4` search yielded multiple
matching visible rows, which allowed the cap to be reached without scrolling or
provider execution.

The tenth selected row exposed:

```text
reference-example-ref_45:
  enabled=true
  selected=true
  help=Remove this reference example
  value begins Selected. Image available.
reference-examples-selection-summary:
  value=10/10
```

After reaching the cap, the probe searched `ref_11`. The live row exposed the
selection-limit state:

```text
workspace-search-search-examples:
  value=ref_11
reference-example-ref_11:
  role=AXButton
  description=Reference example ref_11
  enabled=false
  help=Selection limit reached
  value begins Selection limit reached. Image available.
reference-examples-selection-summary:
  value=10/10
```

## Validation

| Validation | Result | Interpretation |
|---|---|---|
| Reference panel discovery | Passed | `reference-examples-panel` was present and enabled in native Prompt Studio. |
| Initial selectable row state | Passed | Visible rows exposed stable `reference-example-*` identifiers and `Not selected. Image available.` values. |
| Selected row state | Passed | Pressing `ref_1` changed the summary to `1/10` and row state to `Selected. Image available.` with removal help. |

| Ten selected references | Passed | The summary reached `10/10`; the tenth selected row exposed `selected=true` and removal help. |

## Remaining Limitations

- This is an AX probe, not full manual VoiceOver speech-output traversal.
- Running-disabled reference-row state was not live-probed because the smallest
  safe row probe intentionally avoids starting a run. Source-level running hints
  remain covered by `EV-20260622-031`.
- The exact selected set used `ref_40` through `ref_45` after the first four
  examples because the filtered list exposed those rows without needing a scroll
  gesture.
- The `reference-examples-clear-selection` control was not re-probed after the
  final filtered `ref_11` view because it was not present in the visible AX tree
  at that point. Earlier state showed summary transitions and the cap behavior;
  full keyboard traversal remains a separate gate.
- Settings Increased Text Size remains a manual/system-setting-dependent gap;
  no undocumented global text-size defaults were written.
