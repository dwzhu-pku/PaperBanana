## Procedure

The app repository default was temporarily set to the native integration
checkout so the installed app would scan the current branch's ignored local
`results/` diagnostics. The previous default was restored after the probe.

Run Details launch:

```bash
defaults write local.paperbanana.gui settings.repoPath \
  "/Users/jeff/Codex_projects/PaperBanana-native-integrated"
defaults write local.paperbanana.gui paperbanana.intent.destination runDetails
open -a /Applications/PaperBanana.app
```

Provider Run Ledger launch:

```bash
defaults write local.paperbanana.gui paperbanana.intent.destination runLedger
open -a /Applications/PaperBanana.app
```

Both windows were inspected at `1680x970` using a transient Swift AX script
that traversed `AXUIElementCreateApplication(pid)` and matched role,
identifier, title, description, and value fields. The script was run through
stdin and was not added to the repository.

The app default was restored after the probe:

```bash
defaults write local.paperbanana.gui settings.repoPath \
  "/Users/jeff/Codex_projects/PaperBanana"
defaults delete local.paperbanana.gui paperbanana.intent.destination
```

## Limitations

- This probe validates AX exposure of the two table surfaces and their selected
  summaries. It is not a full VoiceOver traversal.
- Run Details defaulted to the `Needs Attention` filter. This probe used local
  ignored diagnostic rows that included needs-attention entries, so rows were
  available. A separate manual pass should still verify filter controls and
  keyboard traversal across all scopes.
- Provider Run Ledger defaulted to `All`, so rows were available from local
  ignored provider-audit diagnostics.
- No screenshots were captured in this evidence item.
- No live provider calls, provider credentials, prompt bodies, raw provider
  payloads, or environment variables were used or printed.
