## Integration Summary

PR #74 was integrated after credential isolation, legacy plot/figure-size fixes,
and hosted plot-code containment.

Conflict and policy handling:

- `README.md` conflicted in the first #74 commit. The resolved version keeps the
  existing native-first/security wording, adds the support-document link, and
  replaces stale "enter your API key" hosted guidance with server-operator
  provider configuration wording.
- `docs/SUPPORT.md` originally said users could restart or click the app's
  key-apply control after key rotation. The integrated version removes that
  stale control reference and says local operators restart the local app/server,
  while hosted users must rely on the server operator.
- `tests/test_docs_contract.py` now rejects stale hosted key-entry guidance:
  `enter your API key`, `click the app's key-apply control`, and `Apply Keys`.
- `agents/polish_agent.py` conflicted in the second #74 commit. The resolution
  preserves both existing `image_size_from_data(data)` behavior and the new
  local/OpenAI-compatible text-route rejection before image generation.

Functional behavior integrated:

- `local/<model>` and `ollama/<model>` route text calls to an OpenAI-compatible
  local endpoint.
- `ollama/<model>` uses `http://localhost:11434/v1` by default when no local
  base URL is configured.
- Local/Ollama model prefixes are explicitly rejected for image-generation
  routes in Vanilla, Visualizer, and Polish agents.
- `configs/model_config.template.yaml` includes `local_openai.base_url` and
  `local_openai.api_key`.
- `docs/SUPPORT.md` documents HF public artifacts, provider quota/billing,
  suspension, local/Ollama text-only limits, third-party relay caveats, and
  API-key rotation.
