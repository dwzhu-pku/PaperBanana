## Integration Summary

PR #73 was integrated after provider/local route support and Planner metaphor
mode. One conflict occurred in `agents/polish_agent.py`; the resolved import
section preserves:

- task-specific style-guide helpers from WP-104;
- `image_size_from_data` from WP-101; and
- local/Ollama image-route rejection from WP-102.

Functional behavior integrated:

- `ExpConfig.agentic_critic` defaults to `False`.
- `--agentic-critic` is exposed in `main.py`.
- `configs/model_config.template.yaml` includes `defaults.agentic_critic`.
- `ExpConfig` rejects `agentic_critic=True` unless `main_model_name` is a
  native Gemini model because the mode uses Gemini `code_execution`.
- Critic and Stylist load task-specific diagram/plot style guides through
  centralized helpers.
- `max_critic_rounds=0` bypasses Critic loops without rerunning Visualizer.
- Agentic Critic stores response text, executable code, and code-execution
  result evidence in the returned run data.
- Default Critic behavior remains the normal router path.
