## Implementation Summary

This commit makes the native validation surface portable enough for review and
CI preparation without relying on a hardcoded developer-machine path.

Added:

- `.github/workflows/python-tests.yml`, an automatic Python 3.12 workflow that
  installs `requirements.txt`, checks diff whitespace, and runs the Python test
  suite without provider secrets.
- `.github/workflows/native-structural.yml`, an automatic macOS structural
  workflow that installs XcodeGen, restores the locked Ruby `xcodeproj`
  dependency through Bundler, and runs source-control/project drift/shell
  syntax checks.
- `.github/workflows/native-xcode27-full-gate.yml`, a manual self-hosted Xcode
  27 full gate for runners labeled `self-hosted, macOS, ARM64, xcode-27`.
- `docs/CI.md`, documenting automatic gates, the explicit Xcode 27 gate, local
  equivalents, secret policy, and `CODEX_XCODE27_BIN`.
- `tests/test_ci_contract.py`, covering workflow presence, policy docs, and
  absence of required `/Users/jeff/.codex/bin/codex-xcode27` paths in native
  gate scripts.

Updated:

- `script/test_all.sh` and `script/xcode27_baseline_guard.sh` now resolve
  `codex-xcode27` from `CODEX_XCODE27_BIN` or `PATH`, and fail with an
  actionable message if unavailable.
- `script/xcode27_baseline_guard.sh` respects externally supplied
  `DEVELOPER_DIR`, defaulting to the expected Xcode beta path.
- `script/check_native_source_control_contract.sh` now treats the workflow
  files, CI docs, and CI-contract test as durable tracked release-support files.

## Limitations

- The GitHub workflows have not run on GitHub because this branch has not been
  pushed in this turn.
- The full Xcode 27 workflow requires a configured self-hosted runner with the
  documented labels; hosted macOS runners are used only for structural checks.
- Dependencies remain unpinned in `requirements.txt`; the clean Python 3.12
  validation proves a point-in-time install, not long-term dependency
  reproducibility.
- Live provider, hosted, visual/accessibility, benchmark, release install, and
  rollback gates remain outside WP-005.
